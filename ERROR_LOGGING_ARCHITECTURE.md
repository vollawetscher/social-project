## Key Components

### 1. ErrorLogger Service
- **Location**: `lib/services/error-logger.ts`
- **Purpose**: Centralized error capture with context
- **Methods**:
  - `log()` - Log with full context
  - `logFromRequest()` - Log from HTTP request
  - `resolve()` - Mark error as resolved
  - `getErrorPatterns()` - Query for AI analysis

### 2. BugReporter Component
- **Location**: `components/error/BugReporter.tsx`
- **Purpose**: User-friendly bug submission UI
- **Features**:
  - Description field
  - Reproduction steps
  - Auto-context capture
  - Reference ID display

### 3. AISupportService
- **Location**: `lib/services/ai-support.ts`
- **Purpose**: AI-powered error analysis
- **Methods**:
  - `analyzeCaseErrors()` - Case-level analysis
  - `analyzeSessionErrors()` - Session-level analysis
  - `analyzeSystemErrors()` - System-wide trends
  - `generateSupportMessage()` - User-friendly response

### 4. API Endpoints
- **`POST /api/error-logs`** - Submit error log
- **`GET /api/error-logs`** - Query error logs
- **`POST /api/support/analyze`** - AI analysis
- **`GET /api/support/system-analysis`** - System trends

## Security & Privacy

### Row Level Security (RLS)
```sql
-- Users see only their own errors
WHERE user_id = auth.uid()

-- Admins see all errors
WHERE role = 'admin'

-- Service role can write all
TO service_role
```

### Data Protection
- IP addresses captured but not displayed to users
- Stack traces only visible to admins
- Case ID less revealing than session ID
- Metadata sanitized of PII

## AI Analysis Capabilities

### Pattern Detection
```
Input: 50 errors across 5 sessions in case ABC
Output:
  - 40 errors: "Speechmatics API timeout"
  - 10 errors: "Invalid audio format"
  
Root Cause: Audio files > 1 hour timing out
Recommendation: Implement chunking for long files
```

### Trend Analysis
```
Input: System errors over 7 days
Output:
  - Spike in errors on weekends (80% increase)
  - Most failures at /api/sessions/[id]/transcribe
  - 60% related to audio processing
  
Trend: Load-related API timeouts during peak usage
Recommendation: Scale infrastructure, add queue system
```

### Contextual Insights
```
Input: Case with 3 sessions, 12 errors
Output:
  - All errors in Session 2 only
  - All during transcription phase
  - All related to large file sizes
  
Context: User uploaded 2-hour meeting recording
Recommendation: Add file size limits and warnings
```
